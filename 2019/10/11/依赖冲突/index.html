为什么会依赖冲突 ？
  • 依赖的包有不同的版本
如何解决冲突
  • 删除重复的依赖
  • 或者修改版本

网络上资料很多，这里就不一一介绍了

在实际工作中碰到了特殊情况

  1. 使用了甲 SDK 中的刷卡 ( 借记卡，信用卡 )

    1. 现在接入乙 SDK 中的读卡 ( 预付卡 )

    2. 甲 SDK 和乙 SDK 模块重复，导致依赖冲突，而且都不能修改

查询了很多资料，最后用修改包名方法解决，可能有点笨

#####下载工具 jarjar.jar

下载地址：http://code.google.com/p/jarjar/downloads/list

#####举个🌰

  • 同级目录下新建 txt
  • 放入需要修改的 jar 和工具包
  • txt 内容 rule sunmi.paylib.** sunmi.paylib.test.@1 ( 要修改的包名 )

#####操作流程

  • 进入到文件目录

    • 输入命令 java -jar jarjar-1.4.jar process jar.txt test.jar test-1.jar
