日常开发总结-OkHttp 源码解析（拦截器链）

发表于 2020-04-01
字数统计: 958 字 | 阅读时长 ≈ 5 分钟

OkHTTP 中同步请求和异步请求获取 Response 对象都会调用到 getResponseWithInterceptorChain() 这个方法，这个方法其实是构成了一个拦截器的链，下面具体看一下方法。

首先可以看到创建了一个 RealInterceptorChain 对象，然后将拦截器集合放入构造函数中完成对象的创建。然后执行 chain.proceed 方法进行请求。下面看下 proceed 方法。

它调用的是另外一个重载方法

它和之前创建 RealInterceptorChain 不同的地方在于传入的索引不一样，这样就把我们的拦截器构成了一个链条。

具体我们可以看看 RetryAndFollowUpInterceptor 这个拦截器。

它执行了 realChain.proceed 方法，上面我们分析了 proceed 方法其实是执行下一条拦截器的 intercept 方法。

#####我们可以简单的理解 : 一个 OkHttp 的请求过程，就是一个一个拦截器执行 intercept 的过程。

#####intercept 方法大概可以理解成

  • 处理 Request 请求
  • 调用下一个拦截器获取 Response
  • 返回 Response 给上一个拦截器
